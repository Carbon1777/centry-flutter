# ТЗ: Список заблокированных пользователей + разблокировка

## Контекст

Блокировка пользователя уже работает: из приватного чата можно удалить чат и заблокировать (`delete_private_chat_and_block_v1`). Запись создаётся в таблице `user_blocks`. Но нет UI для просмотра заблокированных и разблокировки.

---

## Серверная инфраструктура (уже есть)

### Таблица `user_blocks`
| Колонка | Тип | Описание |
|---------|-----|----------|
| id | uuid | PK |
| blocker_user_id | uuid | Кто заблокировал |
| blocked_user_id | uuid | Кого заблокировали |
| created_at | timestamptz | Дата блокировки |

### Существующие RPC
- `block_user_v1` — блокировка пользователя
- `get_my_blocks_v1` — список заблокированных (уже есть, проверить формат ответа)
- `delete_private_chat_and_block_v1` — удаление чата + блокировка

### Нужно создать
- `unblock_user_v1(p_user_id uuid, p_blocked_user_id uuid)` — снятие блокировки
  - Удалить запись из `user_blocks`
  - Возвращать `{ok: true}` или `{error: "NOT_FOUND"}`
  - Дружба НЕ восстанавливается автоматически (только блокировка снимается)

## Клиентская часть (нужно создать)

### 1. Экран списка заблокированных

**Расположение:** доступен из настроек профиля (или из экрана профиля — уточнить где вход).

**UI:**
- Список карточек, аналогичный списку приватных чатов по стилю
- Каждая карточка содержит:
  - Аватар заблокированного пользователя (если есть)
  - Ник (display_name)
  - Дата блокировки (formatted, например «Заблокирован 23 марта»)
- Пустое состояние: «Нет заблокированных пользователей»
- Pull-to-refresh

### 2. Модалка разблокировки

**Триггер:** тап по карточке заблокированного пользователя.

**НЕ открывать мини-профиль.** Вместо этого — AlertDialog:
- Заголовок: «Разблокировать пользователя?»
- Тело: «Пользователь «{ник}» будет разблокирован. Вы снова сможете получать от него сообщения и запросы в друзья.»
- Кнопки:
  - «Отмена» — закрыть
  - «Разблокировать» — вызвать `unblock_user_v1`, убрать из списка, показать toast «Пользователь разблокирован»

### 3. Данные

- Репозиторий: создать `BlocksRepository` / `BlocksRepositoryImpl`
  - `getMyBlocks(appUserId)` → вызов `get_my_blocks_v1`
  - `unblockUser(appUserId, blockedUserId)` → вызов `unblock_user_v1`

## Важно

- Архитектурный принцип: **server-first**. Клиент рендерит snapshot от сервера.
- `get_my_blocks_v1` должен возвращать готовые данные (ник, аватар, дата) — клиент не делает дополнительных запросов.
- Если `get_my_blocks_v1` не возвращает ник/аватар — нужно доработать серверную функцию.
