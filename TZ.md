ТЗ: блок «Рейтинг» для проекта Centry
1. Контекст
Работаем над проектом Centry:
* Flutter client
* Supabase / Postgres / RPC / RLS / Realtime
* архитектура server-first
* клиент — экран и кнопки
* сервер считает, агрегирует и отдаёт готовые данные
2. Точка входа
В полном профиле пользователя нужно добавить переход:
* кликабельный текст с иконкой
* расположен под “Мои места”
* в том же визуальном стиле, что и соседние профильные переходы
По тапу открывается экран рейтинга.

3. Что будет на экране рейтинга
От первоначального наброска фиксируем финальную v1-структуру:
2 рейтинга:
1. Рейтинг пользователей по собранным токенам
2. Рейтинг пользователей по активности


4. Общая компоновка экрана
Экран рейтинга делаем как один экран без вертикального скролла.
Layout:
* 2 колонки
* одинаковые по ширине
* левая колонка: Токены
* правая колонка: Активность
Отображение:
* в каждой колонке показываем top 10
* если текущий пользователь не попал в top 10:
    * внизу соответствующей колонки показываем отдельный блок «Ваше место»
Адаптация:
* экран проектируется без скролла
* на маленьких экранах блоки и строки адаптивно масштабируются по высоте, а не уходят в длинную прокрутку

5. Содержимое строки рейтинга пользователя
Каждая строка рейтинга пользователя содержит:
* место
* аватар
* ник
* под ником — город, если он известен
* справа — значение рейтинга
Для токенов:
* значение = количество токенов
* проектируем поле с запасом минимум до 5 знаков
Для активности:
* значение = количество успешных планов, засчитанных пользователю
Город:
* берём из пользовательской локации
* если пользователь не дал доступ к локации или город недоступен:
    * строку города не показываем
Взаимодействие:
* строки рейтинга не кликабельны
* профиль по тапу из рейтинга не открывается

6. Блок «Ваше место»
Если текущий пользователь не попал в top 10 конкретного рейтинга:
* внизу колонки показываем отдельный блок «Ваше место»
* он содержит:
    * место
    * аватар
    * ник
    * город, если есть
    * значение рейтинга
Если пользователь уже внутри top 10:
* отдельный блок не показываем
Этот блок нужен для:
* рейтинга по токенам
* рейтинга по активности

7. Рейтинг по токенам
7.1. Смысл
Это рейтинг пользователей по количеству заработанных токенов.
Важно:
* считаем только заработанные
* траты / списания / пополнения (в будущем) в этот рейтинг не входят - то есть это не рейтинг баланса
7.2. Текущее состояние
Полноценной бонусная системуже есть и уже начисляет токены за действия.
Нужно заложить:
* серверный контракт
* серверные агрегаты / источник данных
* клиентскую колонку
* формат ранжирования
* блок «Ваше место»
7.3. Поведение в v1
На этом этапе рейтинг по токенам реализуется как готовый боевой каркас, в который приходят реальные данные.
То есть:
* архитектуру строим под настоящую систему
* UI колонка существует уже сейчас
* если данных нет — можно показывать прочерк
* но сама структура должна быть готова к начислению

8. Рейтинг по активности
8.1. Что такое активность
Рейтинг активности пользователя считается и по участию, и по созданным им планам.
Пользователь получает +5 к активности за каждый план, который:
* создан этим пользователем
* закрыт успешно
* не удалён системой из-за отсутствия победителей
* не удалён пользователем
* содержит не меньше 3 участников, включая создателя
Пользователь получает +1 к активности за каждый план, в котором:
* Участвовал пользователь
* закрыт успешно
* не удалён системой из-за отсутствия победителей
* не удалён пользователем
* содержит не меньше 3 участников, включая создателя

8.2. Период
Рейтинг считаем:
* за всё время
Не делаем в v1:
* за неделю
* за месяц
* сезонные срезы

9. Правила ранжирования
Для обоих рейтингов:
Основная сортировка:
* по значению DESC
При равенстве значений:
* пользователи получают одно и то же место
* далее идёт пропуск позиций
Пример:
* 1
* 2
* 2
* 4
Стабильный внутренний порядок при равенстве:
* nickname ASC
* затем app_user_id ASC
Это только для стабильного вывода, а не для изменения места.

10. Server-first контракт
Сервер должен быть источником истины для:
* top 10 рейтинга по токенам
* позиции текущего пользователя по токенам
* top 10 рейтинга по активности
* позиции текущего пользователя по активности
Клиент не должен:
* сам считать рейтинги
* сам агрегировать токены
* сам считать успешные планы
* сам определять место пользователя
Клиент только:
* вызывает RPC / получает snapshot
* рендерит 2 колонки
* показывает “Ваше место”, если оно пришло вне top 10

11. Серверная модель рейтинга
Для токенов
Рекомендуемая логика:
* отдельный серверный агрегат заработанных токенов по app_user_id
* рейтинг работает от этого агрегата
* бонусная система по идее просто пишет в этот слой данные
Для активности
Нужно серверно агрегировать:
* количество успешных планов по creator_app_user_id
* с учётом правил:
    * plan closed
    * not system-deleted because no winners
    * not user-deleted
    * participants_count >= 3 including owner

12. Что должен отдавать сервер
Нужен один серверный endpoint / RPC-снимок экрана рейтинга, который возвращает:
Колонка токенов:
* top 10 пользователей
* моя позиция и моё значение, если я вне top 10
Колонка активности:
* top 10 пользователей
* моя позиция и моё значение, если я вне top 10
Данные строки пользователя:
* app_user_id
* nickname
* avatar
* city (nullable)
* place
* score

13. Клиентская часть
13.1. В профиле
Добавить переход:
* текст + иконка
* под “Мои места”
* в стиле проекта
13.2. Экран рейтинга
Новый экран:
* 2 колонки
* одинаковая ширина
* левая: токены
* правая: активность
Каждая колонка:
* заголовок
* список top 10
* при необходимости блок «Ваше место»
13.3. Состояния
Нужны состояния:
* loading
* loaded
* empty / no data
* error

14. Поведение, если данных пока нет
Токены
Поскольку бонусная система ещё не реализована:
* колонка токенов всё равно существует
* данные под неё резервируются
* если данных нет:
    * можно показывать прочерк / пустое значение
    * но layout остаётся боевым
Активность
Если данных нет:
* показываем пустой рейтинг / заглушечные строки по продуктовой логике
* но структура колонки остаётся полноценной

17. Критерии готовности v1
Этап считается готовым, если:
1. В профиле есть переход Рейтинг
2. По тапу открывается экран рейтинга
3. Экран показывает 2 колонки:
    * токены
    * активность
4. В каждой колонке:
    * top 10
    * одинаковая визуальная ширина колонок
    * корректный вывод места и значения
5. Если пользователь вне top 10:
    * виден блок «Ваше место»
6. Строки не кликабельны
7. Город показывается только если он доступен
8. Вся логика рейтинга считается на сервере

17. Рекомендуемый порядок реализации
Этап 1
Серверный фундамент рейтинга:
* модель данных
* RPC
* агрегаты
* заглушка под токены
* реальная агрегация активности
Этап 2
Клиентский вход из профиля:
* строка “Рейтинг”
Этап 3
Экран рейтинга:
* 2 колонки
* top 10
* ваше место
Этап 4
Шлифовка:
* адаптация под маленькие экраны
* состояния loading / empty / error
* визуальная подстройка

18. Стартовый следующий шаг
Так как проект у вас server-first, правильнее начинать с сервера, а не с экрана.
Первый практический шаг:
* исследовать текущую серверную модель:
    * где и как хранятся пользователи
    * где и как можно взять nickname / avatar / city
    * как сейчас определяется закрытый успешный план
    * где хранится creator плана



Клиент оформи красиво, премиально. 1,2 и 3 место чтоб были с значками золото , серебро и бронза и также подсвечивались тестом. Реализация должна быть не наляпистая колхозная и стильная и премиальная.
