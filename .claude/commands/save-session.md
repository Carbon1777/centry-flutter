Сохрани ключевую информацию из текущей сессии в долгосрочную память проекта Centry (NotebookLM).

**Notebook ID:** `83e2a76b-2e04-4280-885e-32fb0b005a19` (Centry — Knowledge Base)

## Что делать

1. **Проанализируй текущую сессию** и выдели только то, что реально стоит запомнить надолго:
   - Принятые решения и **почему** они приняты (мотивация важнее самого решения)
   - Архитектурные изменения / новые паттерны / важные открытия про код или схему БД
   - Что попробовали и что НЕ сработало (чтобы будущая сессия не наступала на те же грабли)
   - Открытые вопросы / blockers / задачи на следующую сессию
   - Новые факты о проекте, которые не выводятся из текущего состояния кода

2. **НЕ сохраняй:**
   - Пошаговый ход разговора и переписку
   - Содержимое файлов (это есть в git)
   - Тривиальные правки без концептуальной нагрузки
   - То, что уже зафиксировано в коде/коммитах/CLAUDE.md

3. **Сформируй краткое резюме** в формате markdown со структурой:

```
# Session YYYY-MM-DD: <короткая тема>

## Контекст
<1-2 предложения: над чем работали и зачем>

## Решения
- <решение> — **Почему:** <мотивация>

## Что попробовали и не сработало
- <подход> — **Причина отказа:** <...>

## Открытые вопросы / TODO
- <...>

## Изменения архитектуры / новые факты
- <...>
```

   Если какая-то секция пустая — пропусти её, не пиши «нет данных».

4. **Запиши резюме во временный файл** `/tmp/centry_session_<YYYY-MM-DD>_<slug>.md` через инструмент Write.

5. **Залей в NotebookLM** через bash:
   ```bash
   export PATH="$HOME/bin:$PATH" && notebooklm source add /tmp/centry_session_<YYYY-MM-DD>_<slug>.md --notebook 83e2a76b-2e04-4280-885e-32fb0b005a19 --title "Session YYYY-MM-DD: <тема>" --json
   ```

6. **Удали временный файл** после успешной заливки.

7. **Сообщи пользователю** одним коротким абзацем:
   - Что именно сохранено (заголовок source)
   - Source ID из ответа NotebookLM
   - Краткий список ключевых пунктов резюме (буллетами, без воды)

## Важно

- Если сессия была пустой / тривиальной / без новых решений — **не создавай source**, скажи об этом честно и предложи отменить сохранение.
- Если в текущей сессии есть **обновления к уже существующим записям** в NotebookLM (например, изменился статус задачи, которая уже описана) — упомяни это в резюме явно, чтобы при чтении было понятно, что это обновление, а не новый факт.
- Дата сессии = сегодняшняя дата (см. поле `currentDate` из контекста).
