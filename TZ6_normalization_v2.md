## Цель сессии

Из сырого `yandex_parse_all_results.json` (11156 мест) сделать чистую нормализованную базу, готовую к заливке в прод. Ни одного мусорного адреса, ни одного битого телефона, ни одного нерабочего сайта.

## Шаг 1: Финальная фильтрация мусора

### 1.1 Повторный прогон стоп-слов
- Прогнать все названия и сайты через обновлённый список стоп-слов
- Проверить пролезшие: киберспорт, компьютерные клубы, массажные, прачечные, спа-салоны, квесты, батуты, кондитерские
- Вывести список подозрительных для ручного ревью

### 1.2 Проверка города в адресе
- Каждое место должно содержать свой город в адресе
- Удалить места из чужих городов (Чебоксары, Йошкар-Ола, сёла)

### 1.3 Проверка координат
- Удалить места без lat/lng
- Проверить что координаты в пределах города (грубая проверка по bbox)

### 1.4 Дедупликация
- По yandex_id (уже сделано при мёрже)
- По normalized(title) + normalized(address) — одно место может быть найдено в разных районах
- При дубле оставить запись с большим rating и review_count

## Шаг 2: Нормализация адресов

### 2.1 Очистка хвостов
Убрать из адреса всё после номера дома/корпуса/строения:
- "этаж 1", "этаж цокольный", "этаж 2"
- "подъезд 3", "вход со двора", "вход с улицы"
- "офис 12", "помещение 3", "павильон"
- Описания маршрута ("рядом с...", "напротив...")

### 2.2 Формат адреса
Канонический формат: `Город, улица/проспект/переулок, дом[, корп. X][, стр. Y]`
- Убрать дубли города ("Москва, ул. Тверская, 1, Москва" → "Москва, ул. Тверская, 1")
- Стандартизация сокращений: ул., просп., пер., наб., бул., ш., корп., стр.
- Слитные формы: "56к2с1" → "56, корп. 2, стр. 1"

### 2.3 Валидация
- Адрес содержит хотя бы улицу и дом
- Если адрес пустой или невалидный → пометить для ревью

---

## Шаг 3: Нормализация телефонов

### 3.1 Формат E.164
- `+7 (916) 249-37-07` → `+7XXXXXXXXXX`
- Убрать скобки, дефисы, пробелы
- `8-800-...` → `+7800...`
- `8 (495)...` → `+7495...`

### 3.2 Валидация
- Длина: ровно 12 символов (+7 + 10 цифр)
- Невалидные → null
- phones_all: нормализовать все, первый → основной

---

## Шаг 4: Нормализация сайтов

### 4.1 Формат URL
- Добавить `https://` если нет протокола
- Убрать `www.` (оставить домен)
- Lowercase
- Убрать trailing слэши и параметры (?utm_...)
- Убрать якоря (#...)

### 4.2 Стоп-сайты (убрать, поставить null)
- delivery, dostavka — доставка
- shop, store, magazine — магазины
- avito, youla, 2gis — агрегаторы
- instagram.com, vk.com, facebook.com — соцсети (не сайт заведения)
- taplink, linktr.ee — линкаггрегаторы

### 4.3 Валидация
- URL парсится без ошибок
- Домен существует (хотя бы синтаксически)

---

## Шаг 5: Нормализация названий

- Trim, убрать двойные пробелы
- Убрать декоративные символы (★, ♦, ●, и т.д.)
- Оставить ё как есть
- Убрать trailing точки, запятые
- Убрать обрамляющие кавычки если есть

---

## Шаг 6: Маппинг категорий

Привести search_type к нашим категориям:
- "Бар" → `bar`
- "Ресторан" → `restaurant`
- "Ночной клуб" → `nightclub`
- "Сауна" → `bathhouse`
- "Кальянная" → `hookah`
- "Караоке" → `karaoke`
