## Шаг 1: Исследование текущего рейтинга

### 1.1 Разобрать как работает рейтинг
- Найти RPC/функцию расчёта рейтинга (vote_place_v1 и формула)
- Понять как `rating` в place_enrichment соотносится с likes_count / dislikes_count
- Понять что такое "дефолтный рейтинг" и как он складывается с пользовательским

### 1.2 Текущая проблема
Сейчас рейтинг набит только лайками, без единого дизлайка — выглядит неестественно. Нужно:
- Пересчитать лайки и дизлайки так, чтобы итоговый рейтинг был ±0.1 от рейтинга Яндекса
- Распределить дизлайки рандомно (не у всех мест одинаково)

### 1.3 Формула пересчёта
Для каждого места с yandex_rating:
- Целевой рейтинг = yandex_rating (из парсинга)
- `rating = likes / (likes + dislikes)` × 5 (или аналогичная текущая формула)
- Подобрать likes и dislikes чтобы:
  - Для 5.0 → только лайки (или 1-2% дизлайков)
  - Для 4.5 → ~10% дизлайков
  - Для 4.0 → ~20% дизлайков
  - Для 3.5 → ~30% дизлайков
  - Рандомизация ±5% от целевого процента
  - Общее количество голосов тоже варьировать (50-500)

## Шаг 3: Обновление place_enrichment

### 3.1 Массовый UPDATE из staging
```sql
UPDATE place_enrichment pe
SET
  website    = sn.normalized_website,
  phone      = sn.normalized_phone,
  rating     = sn.yandex_rating,
  photos     = CASE
    WHEN sn.photo_needs_placeholder THEN pe.photos  -- оставляем текущий плейсхолдер
    ELSE jsonb_build_array(
      jsonb_build_object(
        'storage_key', 'scraped_photos/' || sn.place_id || '.webp',
        'is_placeholder', false
      )
    )
  END
FROM staging_places_normalized sn
WHERE pe.place_id = sn.place_id
  AND sn.processing_status = 'processed';
```

### 3.2 Обновление адресов в core_places
```sql
UPDATE core_places cp
SET address = sn.normalized_address
FROM staging_places_normalized sn
WHERE cp.id = sn.place_id
  AND sn.normalized_address IS NOT NULL
  AND sn.processing_status = 'processed';
```
