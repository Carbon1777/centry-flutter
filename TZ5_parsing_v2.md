# TZ5 — Парсинг Яндекс.Карт v2: бары, клубы, караоке

## Контекст и проблема

Centry — приложение для встреч с друзьями. Пользователь выбирает место (бар, клуб, 
караоке), создаёт план, друзья голосуют. Критически важно чтобы в базе были ТОЛЬКО 
реальные места для встреч — бары, рестораны, клубы, караоке, театры, кино, бани.

### Что пошло не так в v1
Исходная база мест была собрана из разных источников. В TZ1-TZ3 мы парсили 
Яндекс.Карты чтобы обогатить эти места фото/адресами/телефонами. Парсер искал 
по НАЗВАНИЮ места из нашей базы и брал первый результат. Проблемы:

1. **Мисматчи парсинга** — парсер искал "Ломоносов" (бар) → Яндекс нашёл 
   Дворцово-парковый ансамбль Ораниенбаум. Пользователь видит фото дворца 
   вместо бара. Найдено и исправлено 74 таких мисматча.

2. **Магазины вместо баров** — в базе были пивные магазины (Beer Mug, 33 Кеги, 
   BeerGrad), винотеки, магазины купальников (Танцульки), шоурумы кирпича 
   (Greenhof), брендинговые агентства (Wine Depot). Пользователь видит фото 
   полок с бутылками и думает что это бар. Деактивировано 23+ таких мест.

3. **Сети доставки** — Тануки (60 точек), Якитория (41), Кофемания (30) — 
   это не места для встречи с друзьями. Деактивировано 130+ сетевых точек.

4. **Дупликаты** — одно место размножено по 20-60 раз (разные area_id). 
   Евразия: 62 записи одного адреса. Деактивировано 1301 дупликат.
   Добавлен UNIQUE INDEX на (title, address) WHERE is_active = TRUE.

5. **Школы, спортзалы, детские театры** — Балет Москва (балетсдвухлет.рф), 
   Матчпоинт (спортзал с волейболом), 15 детских театров. Деактивировано 30+.

### Итог чистки
- Было: 9428 мест → Осталось: **6110 активных**
- Убрано: 3318 (мисматчи + магазины + сети + дубли + школы + закрытые)
- 435 мест переименованы на актуальное название Яндекса

### Почему решено парсить заново
Чистка показала что исходная база содержала ~35% мусора. Добавлять места нужно 
ПО-ДРУГОМУ — не по названию из сомнительной базы, а напрямую из Яндекс.Карт 
по рубрикам/запросам с фильтрацией на входе.

## Задача

Написать скрипт парсинга Яндекс.Карт v2, который:

### 1. Сбор данных
- Парсит по **рубрикам** Яндекса: "Бар", "Ночной клуб", "Караоке-бар", 
  "Коктейль-бар", "Лаунж-бар", "Пивной ресторан", "Винный бар"
- Альтернативно/дополнительно — по пользовательскому запросу: 
  "куда сходить вечером в пятницу выпить и повеселиться с друзьями"
- По **8 городам**: Москва, СПб, Новосибирск, Нижний Новгород, 
  Ростов-на-Дону, Казань, Сочи, Краснодар
- Собирает: название, адрес, рубрику/категорию, телефон, сайт, рейтинг, 
  кол-во отзывов, фото (2-3 шт)

### 2. Фильтрация на этапе парсинга
- Рейтинг **≥ 4.5** (обсудить: может 4.0 если мест мало — оценить кол-во по порогам)
- Есть **фото ≥ 2**
- Рубрика **НЕ содержит**: магазин, shop, store, школа, студия, фитнес, 
  доставка, аптека, клиника, салон красоты, автосервис
- Сайт **НЕ содержит**: shop, store, dance, fitness, school, delivery, 
  magazine, avto, auto, clinic, salon
- Название **НЕ содержит**: стоп-слова из TZ1 (шаурма, магазин, автосервис и т.д.)

### 3. Скачивание фото
- 2-3 фото на место
- Структура: `scraped_photos_v2/{city}/{place_id}/1.webp, 2.webp, 3.webp`
- Постобработка: резкость +5-10%, насыщенность +5-10% (как в TZ3)

### 4. Staging-таблица
- Создать `staging_places_v2` с полями: 
  yandex_id, title, address, category/rubric, phone, website, rating, 
  review_count, photo_paths, city, status (new/exists/rejected), notes
- Пометить дубликаты (уже есть в core_places по title+address)
- Пометить подозрительные (стоп-слова в названии/сайте)

### 5. Нормализация (как в TZ2)
- Адреса: единый формат "Город, улица, дом"
- Телефоны: E.164 (+7XXXXXXXXXX)
- Сайты: без http/www, lowercase
- Выбор лучшего фото (скрипт select_best_photo.py из TZ2)

### 6. Заливка в базу
- Только места со status='new' после ручного ревью
- INSERT в core_places с правильным area_id (по адресу → район)
- Загрузка фото в Supabase Storage (place_enrichment bucket)
- Обновление place_enrichment таблицы
- UNIQUE INDEX на (title, address) уже стоит — дубли не пролезут
