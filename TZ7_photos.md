## Цель сессии

Для каждого места выбрать одно лучшее фото (без крупных лиц), загрузить на Supabase Storage, назначить плейсхолдеры для мест без фото.

---

## Шаг 1: Отбор лучшего фото

### 1.1 Скрипт select_best_photo.py (адаптация v2)
Исходные фото: `scraped_photos_v2/{city}/{yandex_id}/1.webp, 2.webp, 3.webp`

### 1.2 Детекция лиц
- OpenCV Haar cascade (haarcascade_frontalface_default.xml)
- Крупное лицо = ширина ≥ 5% ширины кадра → фото исключается
- Мелкие фигуры (фоновые прохожие) — допустимо

### 1.3 Скоринг
| Фактор | Вес | Логика |
|--------|-----|--------|
| Текстура (Laplacian) | +4 | Высокая детализация = интерьер |
| Цветовое разнообразие | +3 | Много цветов = живая обстановка |
| Размер файла | +3 | Больше = больше деталей |
| Тёплые тона | +2 | Hue 10-40° = тёплое освещение |
| Контраст | +1.5 | Хороший контраст |
| Яркость | +1 | Средняя 80-180 |
| Насыщенность | +1 | > 50 = яркий интерьер |
| Бонус фото 2 | +0.5 | Яндекс часто ставит интерьер вторым |
| Таймстамп в углу | -5 | Цифры в нижнем правом углу |

### 1.4 Приоритет
1. Интерьер заведения (лучший)
2. Фасад (приемлемо)
3. Еда/напитки (приемлемо)
4. НЕ использовать: фото с крупными лицами

## Шаг 4: Плейсхолдеры

Места без подходящих фото (все с лицами или 0 фото):
- Категорийный плейсхолдер: `categories/{type}/{N}.webp`
- Детерминированный выбор: hash(title + address) % count_files
- Проверить наличие плейсхолдеров для hookah, karaoke, bathhouse
